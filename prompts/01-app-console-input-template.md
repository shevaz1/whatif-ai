## Apps in Toss 콘솔 입력 템플릿 (초기 설정용)

> 이 템플릿은 **앱을 처음 만들 때** 콘솔에 입력해야 하는 정보를 정리하기 위한 것이다.  
> `{{...}}` 안만 실제 값으로 채워서 사용한다.

---

## 1. 앱 정보

- **한국어 앱 이름**: `{{koAppName}}`  
- **영어 앱 이름**: `{{enAppName}}`  

- **appName**: `{{appName}}`
  - 이 값은 미니앱을 구분하는 **고유 ID**이다.
  - `granite.config.ts`의 `appName` 값과 **완전히 동일**해야 한다.
  - 앱 스킴 `intoss://{{appName}}/...` 호출 시 이 값을 기준으로 Apps in Toss에서 미니앱을 찾는다.

---

## 2. 카테고리 (최대 2개)

> 미니앱 홈에서 노출될 카테고리를 선택한다. **최대 2개까지** 선택 가능.
> (예: 음식·음료, 교육, 건강, 교통, 공공·행정, 소셜, 편의, 쇼핑, AI, 비즈니스, 콘텐츠, 여행, 일상, 정보 등)

- **1차 카테고리**: `{{primaryCategory}}`
- **2차 카테고리 (선택)**: `{{secondaryCategory}}`

설명:
- 사용자가 이 서비스를 어떤 상황에서 떠올릴지 기준으로 가장 자연스러운 카테고리를 선택한다.

## 3. 노출 정보

### 3-1. 부제 (0~20자)

- **부제**: `{{subtitle}}`
- 조건:
  - 서비스를 대표하는 **짧은 설명**
  - **비속어, 느낌표(!) 금지**
- 예시:
  - `나에게 맞는 식물을 찾는 진단 미니앱`
  - `이번 주에 꼭 해야 할 일만 모아 정리해 줍니다`

### 3-2. 상세 설명 (0~500자)

> 사용자가 서비스에서 **무엇을 보고**, **어떤 버튼을 누르고**, **무엇을 경험하는지**를 구체적으로 적는다.

작성 템플릿:

1. 사용자는 `{{홈 화면에서 하는 행동 예시}}` 로 서비스를 시작한다.
2. `{{중간 단계에서 보는 정보·누르는 버튼 예시}}` 를 통해 필요한 정보를 입력하거나 확인한다.
3. 그 결과로 `{{사용자가 얻게 되는 경험·결과 예시}}` 를 제공한다.
4. 추가로 `{{설정/부가 기능 예시}}` 에서 알림, 로그인, 데이터 초기화 등을 관리할 수 있다.

- **상세 설명 초안**:
  - `{{detailedDescription}}`

### 3-3. 앱 검색 키워드

- **키워드 목록**: `{{keywords}}`

가이드:
- 사용자가 앱을 검색할 때 떠올릴 만한 단어/문구를 쉼표(,) 또는 줄바꿈으로 구분해 적는다.
- 예:
  - `식물, 반려식물, 식집사, 진단, 추천, 관리, 키우기`

## 4. 앱 로고 (콘솔 등록)

**노출 위치**  
앱 로고는 토스 앱 **전체 메뉴의 미니앱 홈**에 노출되며, **비게임** 미니앱의 경우 **상단 공통 내비게이션 바**에도 사용된다. 작은 크기에서도 자주 보이므로 선명함과 식별성이 중요하다.

> 콘솔에 첨부되는 **앱 로고 가이드.zip** 등 공식 자료가 있으면 반드시 함께 확인한다.  
> (예: Stitch 등 도구 활용 시) https://stitch.withgoogle.com/

### 4-1. 앱 로고 공식 요구사항 (Apps in Toss)

- **선명도**: 작은 화면에서도 **선명하게** 보여야 한다.
- **직관성**: 서비스 특성이 **한눈에** 드러나야 한다.
- **차별·저작권**: 다른 앱과 **혼동되지 않고**, **저작권 문제가 없는** 오리지널 디자인이어야 한다.
- **다크 모드**: **다크 모드**에서도 자연스럽게 보이는 **색상·형태**를 사용한다.
- **파일 형식**: **정사각형 600×600px**, **PNG**. **배경색 필수**, **투명 배경 사용 불가**.
- **토스 리소스 금지**: 토스에서 제공하는 **아이콘·이미지 리소스**를 앱 로고로 사용할 수 없다 (**2차 가공 포함**).

### 4-2. 로고 제작 시 유의 (요약)

| 항목 | 규칙 |
|------|------|
| **캔버스** | **600×600px** 정사각형, **불투명 배경** (투명 PNG 불가) |
| **배경색** | 브랜드 대표 컬러 권장. **흰색·미색(베이지)** 만으로는 대비·다크 모드 대응이 약해지기 쉬우므로 가이드와 상충 시 **공식 zip 기준**을 따른다. |
| **형태** | **단순·굵은 실루엣**, 고대비. 미니앱 홈·내비에서 윤곽이 뚜렷해야 한다. |
| **안전 영역** | 콘솔 가이드에 **원형 마스크** 안내가 있으면, 중요 요소가 **원 안**에 들어가게 배치한다. |
| **금지** | 토스 제공 에셋, 타사 앱·캐릭터 **표절**, 과한 미세 디테일, 투명 배경 |

### 4-3. 로고 제작용 프롬프트 (복사해서 사용)

아래 블록을 복사한 뒤 `{{...}}` 만 채워 이미지 생성 도구에 넣는다. 출력 후 **600×600 PNG·불투명 배경**으로 보내기 전에 픽셀 아트/선명도를 확인한다.

```
앱 스토어용 정사각형 앱 아이콘 600×600px에 맞는 이미지를 만들어 주세요.

- 주제/컨셉: {{앱 이름 또는 서비스 한 줄 설명}} (예: 감정 기록·월간 패턴을 보는 미니앱)
- 가독성: 아주 작은 크기(예: 48px)로 축소해도 실루엣과 색 구역이 선명하게 구분되도록 단순한 형태와 굵은 윤곽.
- 다크 모드: 어두운 배경 위에 올렸을 때와 밝은 목록에서도 자연스럽게 보이도록 배경과 심볼 대비를 충분히 준다.
- 배경: 전체 캔버스를 채우는 불투명 단색 또는 아주 미묘한 그라데이션. 투명 영역 없음. 배경색은 {{브랜드 대표 컬러 또는 컬러명}}.
- 심볼: 원형 안전 영역 안에 주요 그래픽이 들어가게 배치. 원 밖으로 요소가 잘리지 않게 한다.
- 금지: 글자·텍스트·워터마크, 토스 로고·토스 스타일 공식 아이콘 모방, 저작권 있는 캐릭터 복제, 과한 세부 텍스처, 흰색만 있는 배경(가이드에 따름).
```

### 4-4. 입력 예시

- **주제/컨셉**: `감정을 빠르게 기록하고 달력으로 패턴을 보는 일기형 미니앱`
- **배경색**: `딥 바이올렛(#4C3D8F)` 등 다크 UI와도 어울리는 채도 있는 색

### 4-5. 제작 후 확인

- [ ] **600×600px**, **PNG**, **불투명 배경**(투명 없음).
- [ ] 축소해 봐도 **형태·색이 선명**하다.
- [ ] 서비스 성격이 **직관적**이다.
- [ ] **토스 제공 리소스·타사 표절**이 없다.
- [ ] **다크 모드** 화면에서도 어색하지 않다.

## 5. 가로형 썸네일 이미지

**역할**  
썸네일은 토스 앱 **여러 지면**에 쓰이며, 사용자가 서비스에 들어가기 **전**에 가장 먼저 보는 이미지다. **어떤 서비스인지 바로 이해되고**, **긍정적인 인상**을 주도록 만든다.

### 5-1. 규격

| 항목 | 값 |
|------|-----|
| **비율·크기** | **가로형 1932×828px** |
| **형식** | **PNG** |

### 5-2. 썸네일 주의사항 (Apps in Toss)

- **내용**: 미니앱의 **핵심 플레이 화면**이나 **주요 기능**이 드러나는 이미지를 쓴다.
- **화질**: **선명한 고화질**로 제작한다. 작은 이미지를 **확대해 쓰지 않는다**.
- **레이아웃**: **정해진 규격**에 맞춰 제작하고, 빈 공간을 **색·블러로 억지로 채우지 않는다**.
- **배치**: 주요 내용은 **중앙**에 안정적으로 두고, **가장자리에 너무 붙이지 않는다**.
- **표현**: **비속어**, **정치적 표현** 등 편견·논란 소지가 있는 요소를 넣지 않는다.
- **텍스트**: **과도한 텍스트**를 피하고, 필요 시 **로고·짧은 문구**만 적절히 배치한다.
- **로고 사용 시**: **서비스명이 명확히** 읽히는지 확인한다.
- **차별화**: 다른 앱과 **혼동되지 않게** 한다.
- **저작권**: 문제 없는 **오리지널·라이선스 확보** 이미지다.
- **토스 리소스 금지**: 토스 제공 **아이콘·이미지**를 썸네일로 사용할 수 없다 (**2차 가공 포함**).

### 5-3. 가로형 썸네일 제작용 프롬프트 (복사해서 사용)

가로 **1932×828px**(와이드) 비율로 생성·편집한다. `{{...}}` 만 채운다.

```
가로 와이드 앱 프로모 썸네일 이미지를 만들어 주세요. 최종 출력 해상도는 1932×828 픽셀에 맞춘다.

- 화면 구성: {{미니앱 핵심 화면 설명}} (예: 감정 선택·강도 슬라이더·저장 버튼이 보이는 메인 기록 UI, 또는 월간 감정 캘린더가 보이는 화면). 실제 앱 UI와 톤을 맞춘 깔끔한 모바일 앱 목업 느낌.
- 품질: 고해상도용으로 선명한 가장자리, 작은 썸네일을 늘린 듯한 뿌옇음 금지.
- 레이아웃: 중앙~중앙 약간 상단에 주요 UI 클러스터. 좌우·상하 안전 여백(약 8~12% 이상)을 두어 잘림을 방지. 빈 구역은 의도된 단순 배경으로 자연스럽게 채움(무의미한 블러 패치 금지).
- 텍스트: 과다 문구 금지. 필요 시 짧은 헤드라인 1줄과 {{서비스명}} 로고 타입만. 한글 서비스명은 읽기 쉬운 크기.
- 스타일: 긍정적·친근한 색, 비게임 서비스에 맞는 밝기. 다크/라이트 토스 지면 모두 어긋나지 않게 중간 명도 배경 권장.
- 금지: 토스 공식 로고·토스 앱 UI 복제, 타 앱 스크린샷 표절, 비속어·정치적 심볼, 저작권 캐릭터, 저해상도 업스케일 느낌.
```

### 5-4. 제작 후 확인

- [ ] **1932×828px**, **PNG**.
- [ ] **핵심 기능·화면**이 한눈에 들어온다.
- [ ] **업스케일·저화질**이 아니다.
- [ ] **가장자리 여백**이 충분하다.
- [ ] **토스 제공 에셋**을 쓰지 않았다.

- [ ] **앱 로고**: 600×600 PNG, 배경 필수·투명 불가, 다크 모드·선명도·저작권·토스 리소스 금지 조건을 만족한다.
