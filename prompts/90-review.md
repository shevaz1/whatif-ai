# Apps in Toss Review Checklist

## 1. 운영(Operation)
- [ ] 앱 크래시 가능성 없음
- [ ] 네트워크 실패 시 사용자에게 명확한 피드백 제공
- [ ] 예외 상황에서 무한 로딩 없음
- [ ] 로그/디버그 코드 제거 또는 최소화

---

## 2. 기능(Function)
- [ ] 모든 기능이 실제 사용자 플로우 기준으로 동작
- [ ] 잘못된 입력에 대한 처리 존재
- [ ] 중복 요청/중복 클릭 방지
- [ ] WebView 환경 제약 위반 없음

---

## 3. 디자인(Design)
- [ ] TDS 컴포넌트 사용
- [ ] 정렬/간격/가독성 문제 없음
- [ ] 시스템 UI와 이질감 없음
- [ ] 빈 화면/에러 화면도 디자인되어 있음

### 토스 느낌(Toss Look)
- [ ] 동일 화면에서 spacing 스케일이 일관적이다 (8/12/16/20 등)
- [ ] 타이포 계층이 명확하다 (타이틀/서브/바디/캡션)
- [ ] 섹션 구조가 "제목 → 내용 → 액션"으로 통일되어 있다
- [ ] CTA는 1개가 주가 되고, 보조 액션은 약하다
- [ ] 문구가 짧고 확신 있게 쓰였다 (군더더기 제거)

---

## 4. 보안(Security)
- [ ] 민감 정보 하드코딩 없음
- [ ] 클라이언트에 불필요한 정보 노출 없음
- [ ] 외부 요청/리다이렉트 통제
- [ ] 사용자 입력 검증 존재

---

## 5. Granite / WebView 점검
- [ ] `granite.config.ts`의 outdir 확인
- [ ] 빌드 결과물이 올바른 경로에 생성됨
- [ ] WebView 로드 실패 가능성 없음

---

## 6. 최종 판단
- 출시 가능 여부: ✅ / ❌
- 보완 필요 항목 (있다면)
- 다음 릴리즈 개선 포인트
